# Roadmap Update Workflow

Update, create, or reprioritize a product roadmap.

## When to Use
When the user wants to add items, update statuses, reprioritize, shift timelines, or create a new roadmap.

## Workflow

### 1. Understand Current State

If project tracker is connected: pull current roadmap items with statuses, assignees, dates. Identify overdue, at-risk, or recently completed items. Surface items without clear owners or dates.

If not connected: ask the user to describe or paste their current roadmap (any format).

### 2. Determine the Operation

**Add item**: Gather name, description, priority, estimated effort, target timeframe, owner, dependencies. Suggest where it fits based on current priorities and capacity.

**Update status**: Options — not started, in progress, at risk, blocked, completed, cut. For "at risk" or "blocked": ask for the blocker and mitigation plan.

**Reprioritize**: Ask what changed (new information, strategy shift, resource change, customer feedback). Apply a framework from the **roadmap-management** steering if helpful. Show before/after comparison.

**Move timeline**: Ask why (scope change, dependency slip, resource constraint). Identify downstream impacts. Flag items past hard deadlines.

**Create new roadmap**: Ask about timeframe, format preference (Now/Next/Later, quarterly, OKR-aligned), and initiatives to include.

### 3. Generate Roadmap Summary

#### Status Overview
Quick summary: X in progress, Y completed this period, Z at risk.

#### Roadmap Items
For each: name, one-line description, status indicator, target timeframe, owner, key dependencies. Group by timeframe or theme.

#### Risks and Dependencies
Blocked or at-risk items with details. Cross-team dependencies and status. Items approaching hard deadlines.

#### Changes This Update
Items added, removed, or reprioritized. Timeline shifts. Status changes.

### 4. Follow Up

- Offer to format for specific audience (executive summary, engineering detail, customer-facing)
- Offer to draft communication about changes
- If project tracker connected, offer to update ticket statuses

## Output Format

Clear, scannable format. Tables for roadmap items. Status labels: **Done**, **On Track**, **At Risk**, **Blocked**, **Not Started**.

## Tips

- A roadmap is a communication tool, not a project plan. Keep it at the right altitude.
- When reprioritizing, always ask what changed.
- Flag capacity issues early.
- Dependencies are the biggest risk — surface them explicitly.
- When adding something, always ask what comes off or moves.
